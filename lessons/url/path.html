This is a another combined section, much like the domain name which works as a combination of more than one part. The path and file name are just like files on your computer. Everything that follows the hostname is showing you where the file you want to access is located and ultimately ends with the name of the file. This behavior mimics your filesystem and is no different from when you search for a document on your computer.

As you've been navigating between parts of the URL notice how the file name has changed in the address bar. If you did not notice, please take the time to navigate to a separate section and observe this change.

But wait there's more

Did you notice something else about this particular page's URL? It contains some extra information at the end.

?

Click between the different paths and file name to observe this change. This part of a URL is called the query parameter, this is used to pass along information while still using the same page. Clicking through these different parts you will observe most of the content remains the same minus changes to URL highlighting and the content preceding this paragraph. This page has Javascipt that is looking at the query parameters in order to inform it how to populate these two parts of the page. Query parameters are separated from the file name by a ? and are separated from other query parameters by an &.

There is one last part of a URL that is worth covering. Go back to the first paragraph and click on the underlined domain name. You will be taken to the host page but you will not be at the start of the page, instead you will be at the section that talks about domain names. Try navigating using this link and the link you've used to switch between parts. Note the difference in the URL. You should see a trailing # with some text that follows it. This part of the URL is called the anchor and it allows you to jump to specific parts of a page, specified by the text following the #.
